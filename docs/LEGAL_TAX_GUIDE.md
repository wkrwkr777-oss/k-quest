# 💼 K-Quest 법적/세금 완벽 가이드

## ⚖️ 법적 문제 & 세금 처리 방법

---

## 🚨 핵심 답변 먼저!

### Q1: 수익만 다른 사람으로 설정, 법적 문제?
**A: 방식에 따라 다릅니다!**

❌ **문제 있는 방식:**
```
사업자: 정연주
실제 수익자: 다른 사람 (비밀)
세금 신고: 정연주 명의

→ 탈세! 불법!
```

✅ **합법적인 방식 3가지:**
```
방법 1: 고용 계약
방법 2: 업무 위탁 계약
방법 3: 동업 계약 (지분)
```

---

## ✅ 방법 1: 고용 계약 (가장 간단!)

### 구조:
```
사업자: 정연주 (플랫폼 소유자)
직원: 새로운 사람 (운영 담당)
급여: 월급 또는 수수료 100%

→ 완전 합법!
```

### 법적 구조:
```javascript
정연주 (사업자)
  ↓ 고용 계약
새로운 사람 (직원/운영자)
  ↓ 급여 지급
월 수익의 90-100% 지급 가능
```

### 세금 처리:
```
1. 정연주 (사업자):
   - 매출: 전체 수익
   - 비용: 급여 (새로운 사람에게 지급한 금액)
   - 실제 과세: 매출 - 급여
   - 예: 매출 3,000만원, 급여 2,700만원 (90%)
        → 과세 대상: 300만원만!

2. 새로운 사람 (직원):
   - 근로소득: 2,700만원
   - 소득세: 근로소득세 (15-35%)
   - 4대보험: 선택적 (프리랜서 계약 시 불필요)
```

### 계약서 예시:
```
업무 위탁 계약서

갑: 정연주 (사업자)
을: 새로운 사람 (운영자)

조항:
1. 을은 K-Quest 플랫폼의 전체 운영을 담당한다.
2. 갑은 을에게 월 수익의 90%를 지급한다.
3. 을은 매월 운영 보고서를 제출한다.
4. 세금은 각자 부담한다.
```

### 장점:
- ✅ 완전 합법
- ✅ 간단한 계약서
- ✅ 세금 명확
- ✅ 90-100% 수익 이전 가능

### 단점:
- ⚠️ 새로운 사람은 근로소득세 납부
- ⚠️ 정연주는 여전히 사업자 책임

## ✅ 방법 2: 동업 계약 (지분)

### 구조:
```
사업자: 정연주 + 새로운 사람 (공동 사업자)
지분: 정연주 10%, 새로운 사람 90%
수익 배분: 지분대로

→ 완전 합법!
```

### 법적 절차:
```
1. 공동 사업자 등록
   - 세무서 방문
   - 동업 계약서 제출
   - 사업자등록증에 2명 기재

2. 수익 배분
   - 지분대로 자동 배분
   - 각자 종합소득세 신고

3. 책임 분담
   - 지분대로 책임
```

### 세금 처리:
```
총 수익: 3,000만원

정연주 (10%):
- 소득: 300만원
- 종합소득세 신고

새로운 사람 (90%):
- 소득: 2,700만원
- 종합소득세 신고
```

### 동업 계약서:
```
동업 계약서

당사자:
- 갑: 정연주 (지분 10%)
- 을: 새로운 사람 (지분 90%)

조항:
1. K-Quest 플랫폼을 공동 운영한다.
2. 수익은 지분대로 배분한다.
3. 의사결정은 지분대로 투표한다.
4. 탈퇴 시 지분 매각 가능.
```

### 장점:
- ✅ 완전 합법
- ✅ 명확한 지분
- ✅ 투명한 수익 배분

### 단점:
- ⚠️ 공동 책임
- ⚠️ 의사결정 필요 시 협의

## ✅ 방법 3: 법인 설립 (가장 전문적!)

### 구조:
```
법인: (주)케이퀘스트
대표이사: 새로운 사람
이사: 정연주 (선택적)
지분: 새로운 사람 100% (또는 배분)

→ 완전 합법!
```

### 세금 처리:
```
1. 법인 단계:
   - 매출: 전체 수익
   - 법인세: 10-20%
   - 배당: 지분대로

2. 개인 단계 (새로운 사람):
   - 급여: 월급 수령 (근로소득세)
   - 배당: 배당소득세 15.4%

총 세율: 약 25-30%
개인사업자 (40%)보다 10-15% 절세!
```

## 🌍 외국/한국 세금 이중과세?

### ⚠️ 중요: 이중과세 방지!

#### 상황 1: 한국에만 사업자
```
사업자: 한국
수익: 달러
세금: 한국에만 납부 ✅

이유:
- 사업장이 한국
- 한국 세법 적용
- 미국 등 외국 세금 없음
```

#### 상황 2: 미국 결제 (Stripe/PayPal)
```
Stripe/PayPal 수수료: ~3%
미국 세금: 없음! ✅

이유:
- 단순 결제 대행
- 사업자는 한국
- 세금은 한국에만
```

#### 상황 3: 법인이 미국/해외
```
⚠️ 이중과세 가능
- 법인 소재국: 법인세
- 한국 (배당 시): 배당세

→ 비추천! 한국 법인만 설립!
```

### ✅ 결론:
**한국 사업자/법인이면 한국 세금만 내면 됩니다!**

## 💵 결제 통화 (달러/원화)

### Stripe/PayPal 설정

#### 옵션 1: 달러만
```javascript
// 외국인은 달러 결제
payment: {
  currency: 'USD',
  amount: 100, // $100
}

장점:
- 외국인 편리
- 환율 걱정 없음

단점:
- 한국인 불편 (환율 계산)
```

#### 옵션 2: 원화만
```javascript
payment: {
  currency: 'KRW',
  amount: 130000, // 130,000원
}

장점:
- 한국인 편리

단점:
- 외국인 환율 혼란
```

#### ✅ 옵션 3: 자동 통화 선택 (추천!)
```javascript
// 사용자 위치/언어에 따라 자동
function getPaymentCurrency(userLocation, userLanguage) {
  if (userLocation === 'KR' || userLanguage === 'ko') {
    return {
      currency: 'KRW',
      symbol: '₩',
    };
  } else {
    return {
      currency: 'USD',
      symbol: '$',
    };
  }
}

// 실시간 환율 적용
const krwAmount = usdAmount * exchangeRate;
```

#### ✅ 옵션 4: 사용자 선택 (최고!)
```javascript
// UI에서 선택 가능
<select>
  <option value="USD">$ USD</option>
  <option value="KRW">₩ KRW</option>
  <option value="EUR">€ EUR</option>
  <option value="JPY">¥ JPY</option>
</select>

장점:
- 최고의 UX
- 모든 통화 지원
```

### Stripe 다중 통화 설정:
```javascript
// Stripe는 135+ 통화 지원!
const paymentIntent = await stripe.paymentIntents.create({
  amount: amount,
  currency: selectedCurrency, // USD, KRW, EUR, JPY...
  payment_method_types: ['card'],
});
```

### 환전 수수료:
```
Stripe:
- 같은 통화: 수수료 없음
- 다른 통화: ~1% 환전 수수료

예:
- USD → USD: 무료
- USD → KRW: 1%
- KRW → USD: 1%
```

## 🏦 정산 통화

### 사업자 계좌 설정

#### 한국 사업자:
```
Stripe 계좌: 한국 원화 계좌
정산: 원화로 자동 환전

예:
- 외국인 결제: $100 (USD)
- Stripe 환전: ₩130,000 (자동)
- 계좌 입금: ₩130,000
```

#### 달러 계좌 원하면:
```
1. 외화 통장 개설
2. Stripe에 달러 계좌 등록
3. 달러 그대로 입금

장점:
- 환율 변동 대응
- 달러 자산 보유

단점:
- 세금 신고 시 원화 환산 필요
```
