# 🚀 K-Quest 로컬 임시 웹 실행 가이드

## ⚠️ 중요: 이것은 로컬 테스트용입니다
- **quest-k.com에는 절대 배포되지 않습니다**
- 모든 기능은 **http://localhost:3000**에서만 작동합니다
- 안전하게 테스트하고 확인하세요!

---

## 📋 1단계: 환경 변수 설정 (필수)

1. `.env.example` 파일을 복사하여 `.env.local` 생성:
   ```bash
   copy .env.example .env.local
   ```

2. `.env.local` 파일을 열고 Supabase 정보 입력:
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-key-here
   ```

   **Supabase 키 찾는 방법:**
   - https://supabase.com 로그인
   - 프로젝트 선택
   - Settings → API 메뉴
   - URL과 anon/public 키 복사

---

## 🚀 2단계: 로컬 서버 실행

```bash
npm run dev
```

서버가 시작되면:
```
✓ Ready in 3.2s
- Local:   http://localhost:3000
```

## 🌐 3단계: 브라우저에서 확인

### 📱 주요 페이지 (모두 localhost에서만 작동)

| 페이지 | URL | 설명 |

## ✅ 4단계: 기능 테스트

### A. 회원가입 테스트
1. http://localhost:3000/auth/signup 접속
2. 이름, 이메일, 비밀번호 입력
3. "Create Account" 클릭
4. 이메일 확인 (Supabase에서 확인 메일 발송)

### B. 로그인 테스트
1. http://localhost:3000/auth/signin 접속
2. 이메일, 비밀번호 입력
3. "Sign In" 클릭
4. 자동으로 대시보드로 이동

### C. 퀘스트 기능 테스트
1. http://localhost:3000/quests 접속
2. 검색, 필터 기능 확인
3. 퀘스트 카드 클릭하여 상세 페이지 이동
4. "Post a Quest" 버튼으로 새 퀘스트 생성

## 🛑 문제 해결

### Q1: "Cannot find module '@supabase/ssr'" 에러
```bash
npm install @supabase/ssr @supabase/supabase-js
```

### Q2: 로그인이 안 돼요
- `.env.local` 파일에 Supabase 키가 올바른지 확인
- 서버 재시작: `npm run dev`

### Q3: 페이지가 비어있어요
- 서버가 실행 중인지 확인
- 브라우저 콘솔에서 에러 확인 (F12)
